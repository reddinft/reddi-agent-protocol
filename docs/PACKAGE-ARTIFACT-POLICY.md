# Package Artifact Policy

Tracked source should stay separate from local package install/build output.

## Policy

- Do not commit package-local `node_modules/` directories.
- Do not commit `packages/openrouter-specialists/dist/`; it is generated by `npm test` / `npm run build` inside that package.
- `packages/x402-solana/dist/` is currently still tracked for compatibility with existing `file:../x402-solana` consumers and should be revisited when the repo has a proper npm workspace or package build orchestration.

## Current validation order

When validating OpenRouter specialist changes from a clean checkout:

```bash
cd packages/x402-solana
npm install --ignore-scripts
npm test
npm run build

cd ../openrouter-specialists
npm install --ignore-scripts
npm test
npm run validate:wallet-manifest
```

The devnet balance checker intentionally exits non-zero while wallets are below the configured threshold:

```bash
npm run check:devnet-balances
```

## Follow-up

Longer term, convert the repo to explicit npm workspaces or add root scripts that build package dependencies in order. Once consumers no longer need committed x402 build output, remove `packages/x402-solana/dist/` from git as well.
